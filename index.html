計測データの読み方

画面上部に表示される各指標は、現在の測位状態を評価したものです。

• 品質スコア: 精度を5段階（S:最高、A:高、B:実用、C:低、D:参考）でランク付け。
• ◎ ±XXm: 推定される誤差の半径。値が小さいほど正確です。
• ⏱ XX.Xs: 測位を開始してからの経過時間。
• 📡 Xサンプル: 計算に使用された観測データの蓄積数。
• ✦ KF +XX%: フィルタ処理によって、デバイスの生データからどれだけ精度が改善したかを示す指標です。
• 精度推移グラフ: 誤差が時間とともに「収束」していく様子を可視化しています。

idokeido とは

「idokeido（イドケイド）」は、ブラウザが持つ位置情報取得（Geolocation API）の能力を最大限に活用し、より正確で信頼できる座標を導き出すためのツールです。

スマートフォンのGPSには、建物の反射やノイズが常に含まれています。本アプリはそれらのノイズを数学的なフィルタで取り除き、単なる「点」ではなく、データに基づいた「確かな位置」を表示することを目指しています。

高精度化のしくみ（カルマンフィルタ）

本システムは、2次元カルマンフィルタという、航空宇宙や自動運転にも使われるアルゴリズムを採用しています。

デバイスから送られてくる「生の座標」には常に揺らぎ（ノイズ）があります。フィルタは、移動速度や測定誤差の大きさをリアルタイムに計算し、突発的な座標の「飛び」や「ジグザグ」を抑制。実際の物理的な動きに沿った滑らかな座標へと補正を行います。

技術詳細・数学的基盤の表示

1. 座標系と状態定義

地球上の緯度経度を局所的な平面直交座標系（メートル単位）に射影し、状態ベクトルを x = [px, py, vx, vy]T と定義。位置だけでなく「速度」を推定することで、慣性を考慮した滑らかな推移を実現しています。

2. 予測と更新のループ

Predict:
xk|k-1 = F·xk-1|k-1 （運動モデルによる予測）
Pk|k-1 = F·Pk-1|k-1·FT + Q （不確実性の増大）

Update:
K = Pk|k-1·HT(H·Pk|k-1·HT + R)-1 （カルマンゲイン）
xk|k = xk|k-1 + K(zk - H·xk|k-1) （観測値による補正）
Pk|k = (I - K·H)·Pk|k-1 （不確実性の圧縮）

3. ノイズパラメータの動的最適化

観測ノイズ行列 R にはOSが提示する「Accuracy（期待精度）」を2乗した分散値を代入。プロセスノイズ Q には微小な加速度成分（約0.1m/s²）を見込むことで、静止時の細かな揺れ（ハンチング）を抑えつつ、移動開始時の追従性を両立させています。

※計算には JavaScript による 4×4 行列演算ライブラリを自前実装し、ブラウザメインスレッド上で低遅延実行しています。

測位の収束と判定プロセス

測位を開始した直後は、数千メートルの大きな誤差が出ることがありますが、しばらく待つことで精度が徐々に高まっていきます（収束現象）。

本アプリでは、推定誤差が一定以下（10m以内）になるまで継続的に観測を行います。精度が十分に「収束」したと確信できたタイミングで、ボタンを押して結果を確定させてください。もし精度がなかなか上がらない場合は、空が広く見える場所へ移動し「再取得（Retry）」を試みるのが効果的です。

プライバシーとiOSでの制約

プライバシー保護: 測位や計算処理はすべてあなたのブラウザ内で完結しており、位置情報が外部サーバーへ送信されることはありません。

iOSの精度制限: iOSで「正確な位置情報」がオフになっている場合、OSによって意図的に誤差が加えられます。この状態では品質スコアが低く表示され、システムから警告が通知されます。

外部GNSS連携（試験的）

Web Serial/Bluetooth APIを介して、高感度外部GNSSレシーバー（u-blox製等）から直接データを読み取ります。

ブラウザ標準の位置情報機能をバイパスし、生のNMEAデータを解析することで、衛星の配置状況（HDOP）やRTK測位の状態など、プロフェッショナルな情報を表示します。良好な条件下ではセンチメートル級の精度を追求することも可能です。
